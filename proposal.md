# 面向多镜头生成视频的场景自一致性诊断 Benchmark Proposal

## 1. 背景与动机

真实影视创作并不是由彼此独立的单镜头组成。一个场景通常通过多个镜头完成一段连续叙事：远景建立空间，近景呈现人物反应，特写突出关键道具，正反打推进对话，插入镜头补充动作细节，再回到远景重新确认空间关系。在这个过程中，镜头的景别、机位和构图不断变化，但观众默认这些镜头发生在同一个连续的时空中。

因此，影视创作对"连续性"有天然要求。同一场景中的人物外观、服装、道具状态、背景陈设、空间布局和光影条件，都需要在剪辑之间保持自洽。一个杯子被放在桌上后，后续镜头中不应无故变色或消失；墙上的画、桌边的灯、窗户和门的位置关系不应在正反打之间随意变化。这些问题在传统影视制作中被称为 continuity errors，也就是穿帮或连续性错误。

随着视频生成模型从单镜头短片段走向多镜头叙事生成，它们也开始面对类似要求：模型不仅要让每个镜头都合理、清晰、好看，还要让多个镜头组合起来像发生在同一个连续场景中。仅评估单镜头质量或主体一致性已不足以衡量这种能力——一个视频即使主角始终一致，也可能由于道具、背景、空间或光影的不连续而破坏叙事可信度。

因此本 benchmark 的核心出发点是：

**主角一致不等于场景连续。**

我们要评估的是：生成模型能否在一个较长的多镜头场景中，保持完整场景的影视连续性。

## 2. 核心问题定义

### 2.1 我们只测"自一致性"，不测"符不符合某个标准答案"

真实评测场景有两个硬约束：

1. **Benchmark 里只有 video 和 caption（prompt），没有任何其它先验标注。** 我们不假设存在一份逐元素声明"什么该保持"的连续性契约，也不假设有人预先标注了每个道具的正确颜色、位置。
2. **评测过程完全自动，没有人工参与。**

caption 只约束"要生成什么"，但它无法穷尽画面里的一切——杯子什么颜色、墙上挂什么画、光从哪来，大量细节由模型在 prompt 之外**自由发挥**，而我们无法约束这种发挥。因此"本该是什么"根本无从定义。

我们把评测目标收敛为一个不依赖外部标准答案、也不依赖人工的量：

**多镜头场景内在自一致性 (Intrinsic Multi-shot Self-Consistency)：模型自由生成的视觉元素，一旦在某个镜头中被确立，就应在同一场景的后续镜头中与自身保持自洽；若发生无剧情依据的变化，即构成连续性错误。**

关键在于：**参照系是视频自己，而不是任何外部契约。** 某元素在其首次/前序出现时的状态，就是它自身后续镜头的"事实基准"。prompt 说"书房里有个杯子"，模型自己决定它是蓝色——没人要求它必须蓝，但一旦第一镜生成了蓝杯子，第四镜变成白杯子就是自相矛盾。

### 2.2 与 prompt-following 解耦

需要明确区分两个概念，本 benchmark 只做后者：

- **Contract / Prompt Fidelity（不评）**：模型有没有生成 prompt 要求的元素。prompt 要蓝杯子而第一镜没有蓝杯子，是 fidelity 问题。
- **Intrinsic Continuity（本 benchmark 评）**：已经生成出来的元素，在后续镜头中是否与自身保持自洽。第一镜蓝杯子、第四镜白杯子，是 continuity 问题。

这个解耦让评测独立于 prompt 兑现程度，只考察跨镜头的自洽能力。

## 3. Benchmark 定位与贡献

本 benchmark 定位为：

**一个面向多镜头生成视频的、仅凭 video+caption 的全自动场景自一致性诊断 benchmark。**

目标不是给模型一个笼统的 consistency score，而是像影视 continuity supervisor 一样，指出生成视频中具体哪里、哪个元素、在哪些镜头之间发生了穿帮。四点贡献：

### 3.1 新任务：caption-only、全自动的多镜头自一致性
已有评测大多依赖参考视频、逐元素标注或人工判分。我们把任务形式化为**只有 prompt 和生成视频、以视频自身为参照、无人工介入**的自一致性诊断，并把评测目标从 subject consistency 扩展到 whole-scene continuity（主体、服装、道具、背景陈设、空间关系、光影）。

### 3.2 可复现的分解式评测器（经典算法 + 专用模型，而非 MLLM 黑箱）
我们不把整段视频丢给一个 MLLM 打分，而是把评测**分解成可复现、可审计的模块**：经典 shot 检测 → 主体 grounding + 外观/人脸 embedding → **几何法空间视角分组** → 组内结构/几何/道具比较，全程带**视角门控**。其中"空间视角分组"是一个不依赖 MLLM 的几何模块（局部特征匹配 + RANSAC），既提供可复现的分组，也提供跨镜头比较所需的视角关系（同机位强比、正反打保守）。MLLM 只在几何失效的暗糊近景处作为**兜底**，且其分组结果可被几何证据审计。

### 3.3 抗刷分的指标：richness × self-consistency 二维 leaderboard
纯自一致性有一个致命陷阱：**画面越空洞、能自相矛盾的东西越少、越容易得高分**，从而反向激励模型生成贫乏画面。我们因此**不排单一分数、也不排 finding 数量**，而是在二维上排：一轴是模型自己 commit 了多少可追踪元素（Scene Richness），一轴是这些元素的跨镜自洽率（Self-Consistency Rate）。空洞→richness 低、丰富但乱→consistency 低，只有"又丰富又自洽"才能赢，恰好对应真实影视连续性的要求。

### 3.4 无人工的可信度验证：受控扰动标定
既然没有人当裁判，指标的可信度靠**构造已知答案**自证：对自洽片段程序化注入已知漂移（改色、删道具、换脸、平移家具），要求对应分数单调下降，并测其检出率。这把每个分数从"经验阈值"变成"被验证的判据"，也把"如何认证一个自动视频连续性指标"本身作为方法贡献。

## 4. 数据构造流程

真实影视视频提供多镜头结构和连续性压力，但不直接作为 ground truth。发布的是原创 episode 的 **prompt(caption) + 生成视频 + 自动诊断结果**，外加一个**独立的扰动验证集**（用于标定评测器，非 benchmark 主数据）。

整体流程：

真实叙事视频
→ 镜头结构与连续性压力点挖掘
→ 抽象改写为原创 episode 的 shot-level prompt
→ 输入生成模型得到多镜头视频
→ 全自动诊断 continuity errors

### 4.1 从真实叙事视频中挖掘结构
真实影视剧、短片、广告中天然存在同场景多镜头叙事（远景建立、近景刻画、特写道具、正反打、插入、遮挡后重现、道具交接、回到远景）。我们挖掘的不是具体人物或画面，而是**多镜头组织方式和连续性压力点**。

### 4.2 抽象改写为原创 episode（只产出 prompt）
为避免版权问题，真实视频只作结构来源。我们替换具体人物、地点、物体和情节，但保留其连续性挑战，写成 shot-level prompt。注意：我们**不再产出逐元素连续性契约**——什么该连续由"同场景静态先验 + 视频自身"隐式决定，评测时不需要它落盘。

### 4.3 生成并诊断
将 episode prompt 输入不同视频生成模型，收集生成视频，按 §7 全自动诊断。最终发布数据：
- 原创 episode 的 shot-level prompt；
- 各模型生成视频；
- 自动诊断结果与分数；
- 评测代码与指标；
- 独立的扰动验证集（自动带标签，用于评测器标定）。

## 5. Episode 组织

每个 episode 是同一场景内的短多镜头叙事，通常 4–8 个镜头。每个 episode 只包含两部分（**没有 continuity contract**）：

### 5.1 Shot-level Prompt
逐镜描述内容、动作、视角、镜头类型与时长。它定义模型要生成什么，也是评测时做 shot 对齐、判定实体应否出现的唯一文本依据。

### 5.2 Evaluation Target（可选、仅供分析分组）
标注该 episode 主要制造的连续性压力（主体持续 / 道具重现 / 背景稳定 / 状态保持 / 空间关系 / 光影氛围），仅用于结果分层分析，不参与判分、不作为标准答案。

## 6. 连续性错误类型

第一版定义五类核心 continuity errors，每类对应 §8 中一组可计算分数：

- **Missing**：应持续存在的元素在后续镜头中消失。
- **Appearance Drift**：元素外观无理由变化（颜色、形状、材质、服装、图案、身份漂移）。
- **State Drift**：元素状态不连续（已拿起的道具无理由回到桌上等）。
- **Spatial Drift**：元素位置或相对关系不合理变化（门、窗、桌、沙发、墙画等空间关系不自洽）。
- **Lighting / Atmosphere Drift**：光源方向、色温、亮度、时间氛围突变。

## 7. 评测方法（分解式、可复现流水线）

评测器判断生成视频是否与自身发生跨镜头矛盾。四个阶段，全部自动，经典算法与专用视觉模型为主、MLLM 仅兜底。（完整算法、参数与分数公式见配套 `pipeline_plan.md`，此处为方法概述。）

### 7.1 Stage 1 — Shot 检测与关键帧
经典内容差分法（HSV 直方图卡方距离 + 自适应阈值）切分镜头。实测发现 AI 生成视频常把分镜渲成**连续运镜**，纯差分对硬切可靠、对软分镜漏检、对快速运镜误检；因此采用 **prompt-anchored 对齐**：把检测峰吸附到 prompt 声明的分镜时间点。每个 shot 取"清晰度−运动"最优的中段帧为关键帧，避开运动模糊。

### 7.2 Stage 2 — 主体 grounding 与背景分离
从每镜 prompt 解析应出现的实体及其 referring expression，用 **GroundingDINO** 检测定位；两个相似主体用**空间座位先验 + 匈牙利匹配**消歧。裁出主体 crop，并把主体框与动态前景（食物等）mask 掉得到背景图，供背景评分。同时记录 prompt 声明"应出现"但未检出的实体，供 Missing 判定。

### 7.3 Stage 3 — 跨镜头自一致比较
分三条线，均带视角门控（只在可比视角间做强比较）：

- **主体外观**：对每个实体的 crop 取 DINOv2 embedding，跨镜两两余弦相似 → 抓 Appearance / State Drift。
- **人物身份**：对人脸取 ArcFace embedding，跨镜两两余弦相似 → 抓身份漂移/换脸；对光照角度比 DINO 更鲁棒，是人物主指标。
- **背景/场景**：先用**几何法做空间视角分组**（局部特征 + RANSAC 单应内点数构亲和图 → 聚类；几何失效处退回全局 embedding 分组并记 confidence），再在**同视角组内**计算结构一致（背景对齐后 SSIM/LPIPS）、几何一致（RANSAC 内点率）、道具一致（道具框计数与位置 IoU）→ 抓背景漂移、结构扭曲、道具凭空增减；三人座位左右次序与相对位置的跨镜一致性 → 抓 Spatial Drift / 位置跳变。

### 7.4 Stage 4 — 汇总与 typed findings
把各分数汇总为分维度与总体分数，并输出元素级、类型化、可定位的 finding：`{element, error_type, affected_shots, evidence(分数与观测), confidence, severity}`。这正是 benchmark 的诊断价值所在。

## 8. 指标设计

### 8.1 二维主结果：Scene Richness × Self-Consistency Rate
不排单一数、不排 finding 数。

- **Scene Richness**：模型自己 commit 的可追踪元素规模（跨镜复现的主体/道具/背景锚点数量）。空洞画面在此暴露。
- **Self-Consistency Rate**：这些元素跨镜自洽的比例。丰富但不自洽的画面在此暴露。

以二维散点/前沿呈现 leaderboard，抗"靠空洞刷分"。

### 8.2 Scene Continuity Score（按机会数归一化的标量主指标）
需要单一排序时，用**机会数归一化**汇总，避免不同富度不可比：
```
SCS = Σ_c (w_c · score_c · opp_c) / Σ_c (w_c · opp_c)
opp_c = 该维度的可比较机会数（实体出现的 shot 对数、同视角组内帧对数等）
```
分项：Subject / Face、Prop、Background-Structure、Spatial、Lighting，均由 §7 的分数派生。

### 8.3 诊断型指标
体现诊断能力：每 episode 的 continuity error 数、每类错误频率、每类元素错误率、不同镜头结构下的错误率、major error rate。用于揭示模型是易丢道具、易破坏空间关系，还是在特定镜头结构中更易失败。

### 8.4 评测器验证（无人工，靠受控扰动）
在**扰动验证集**上标定与验证：注入 k 个已知漂移，要求对应分数单调下降；报每个分数的检出率、误报率、定位准确率，以及阈值 τ 的 ROC 拐点。诚实说明局限：注入的是"干净替换"，与生成模型真实的诡异漂移（半渐变、形变、身份融合）分布不同，扰动集证明"检得出干净错误"，不保证覆盖所有生成怪象。

## 9. 缓解方法（辅助实验）

建议包含一个简单 baseline：**Continuity Memory Prompting**——生成后续镜头时显式维护一个 scene continuity memory，记录前文需保持的元素、属性、状态、空间关系与光影，作为约束加入后续 prompt。其作用不是提出新模型，而是验证 benchmark 能否指导改进：哪些错误可被显式记忆减少、哪些仍难缓解、失败究竟来自记忆不足、空间建模不足还是细粒度视觉控制不足。预期它对主体和显著道具有帮助，但对复杂背景陈设、空间布局、光影连续提升有限——进一步说明本 benchmark 暴露的是当前模型尚未解决的深层问题。

## 10. 总结

真实影视中同一场景往往包含较长的多镜头叙事，天然要求人物、道具、背景、空间和光影在剪辑之间保持连续。随着视频生成走向多镜头叙事，仅评估主体一致性已不够。

我们提出一个**仅凭 video 与 caption、以视频自身为参照、全自动**的多镜头场景自一致性诊断 benchmark：从真实叙事视频挖掘多镜头连续性压力，抽象为原创 episode 的 shot-level prompt；用经典算法与专用视觉模型组成的分解式评测器（含无 MLLM 的几何视角分组），诊断生成视频中与自身矛盾的 continuity errors，输出元素级、类型化 findings；用 richness × self-consistency 二维 leaderboard 抗空洞刷分，用受控扰动在无人工条件下标定指标可信度。

最终，本 benchmark 把多镜头视频评测从"主体是否一致"推进到"整个场景是否像真实影视一样、与自身连续自洽"。
